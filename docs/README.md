# suisoku docs

Vercel 向けの最小構成を配置しています。

入口画面では、ハナビBH と ハッピージャグラーVⅢ と スマスロ Lハナビ と 新ハナビ の4ページに移動できます。ボタンは縦並びで、スマホでも押しやすい間隔にしています。

各画面では、入力欄の名前を左、入力欄を右に並べています。最後に期待値用の条件と店情報の入力欄があります。ハナビBH、スマスロ Lハナビ、新ハナビでは攻略率を独立した入力欄で使い、初期値は75%です。ハッピージャグラーVⅢでは攻略率は使わず、公表値、チェリー狙い、フル攻略の3択をラジオボタンで選びます。見出しの右隣に「期待値の計算に使用」と表示しています。その下の店情報では貸枚数は46、交換率は5.0が初期値です。攻略率の入力欄は短めにしてあり、店情報では貸枚数と交換率の入力欄も短め、現金投資は少し広めにしてあります。貸枚数の右に1枚あたりの貸し単価、交換率の右に1枚あたりの交換単価、現金投資の右に換金ギャップ損をリアルタイム表示します。

推測ボタンを押すと、入力値から実践G数や各確率を計算して結果を表示します。ボーナス確率の推測では、開始前の値を差し引かず、現在のG数、BIG、REGの累計値をそのまま使います。小役やBIG中、REG中などの推測では、ユーザーの実践分だけを分母に使います。BIG中の項目はBIG中の消化G数を分母に使い、REG中の項目はREG中の消化G数を分母に使います。

推測結果では、総合推測の割合を最初に表示し、その下の実践期待値の表で実践G数、機械割、設定1・2・5・6それぞれの期待値、推測割合、推測期待値、およびその合計期待値を表示します。さらに、合計の推測期待値をもとに700Gあたりで換算した期待時給も表示します。期待時給は「期待時給 1399.6円 / h」のように、名前の右隣へ横並びで表示します。機械割の見出し下には攻略率を表示し、機械割はその攻略率に応じて通常時の機械割と完全攻略の機械割の間で変わり、期待値の計算にもその値を使います。円換算は交換率から自動で変わり、現金投資額ぶんの換金ギャップ損も差し引いています。

ハナビBH では、ボーナス、通常時小役、BIG中、花火チャレンジ中、花火ゲーム中について、各項目ごとの推測値を表示します。合算と2役合算は表示だけで、総合推測には含めていません。

ハッピージャグラーVⅢ では、ボーナスと通常時小役のブドウについて、各項目ごとの推測値を表示します。ボーナス合算は表示だけで、総合推測には含めていません。期待値計算は攻略率ではなく、公表値、チェリー狙い、フル攻略の選択によって機械割を切り替えます。ブドウが未入力のときは、ブドウを0個として扱わず、総合推測から外してボーナス確率のみで推測します。

スマスロ Lハナビ では、ボーナス、通常時小役、BIG中、REG中、花火チャレ中、花火ゲーム中について、各項目ごとの推測値を表示します。通常時小役では風鈴Aと風鈴Bをそれぞれ総合推測に使い、風鈴合算は表示だけにしています。ボーナス合算も表示だけで、総合推測には含めていません。REG中は消化G数、1枚役、バラケ目の入力欄を追加しています。

新ハナビ では、ボーナス、通常時小役、BIG中、REG中、REGピース画面、花火チャレ中、花火ゲーム中について、各項目ごとの推測値を表示します。通常時小役では風鈴Aと風鈴Bをそれぞれ総合推測に使い、風鈴合算は表示だけにしています。通常時小役の入力欄には氷A、チェリーA2、チェリーBもあります。ボーナス合算も表示だけで、総合推測には含めていません。REG中は消化G数、1枚役、ハズレの入力欄を追加しています。さらにREGピース画面では、0~4回時、5~6回時、7回以上時ごとに試行回数と発生回数を入力でき、入力欄は3桁程度の短めの幅にしています。画像どおりの発生率を使って総合推測と各項目別推測へ反映します。

各項目ごとの推測値は、下のスペック表と同じまとまりごとの表で整理しています。各項目の表では、1列目の見出しにグループ名を入れ、その下に分母となるゲーム数を表示しています。各列見出しは「項目名」と「実測の数（確率）」の2行で小さく表示しています。

設定表は、機種ごとのまとまりに合わせて分けています。ハナビBH の通常時小役には2役合算を表示し、スマスロ Lハナビ と 新ハナビ の通常時小役には風鈴合算を表示します。新ハナビ の通常時小役には氷A、チェリーA2、チェリーBも表示し、REGピース画面の0~4回時、5~6回時、7回以上時の発生率も別表で表示します。ハッピージャグラーVⅢ では、基本スペック表にBIG、REG、ボーナス合算、公表値を表示し、別表で公表値、チェリー狙い、フル攻略の機械割を並べます。基本スペックには通常の機械割や打ち方ごとの機械割を表示し、確率の分母は小数もそのまま見えるようにしています。

全体の見た目は、スマホで見やすいように横幅と余白を詰めています。スマホでも、2項目と3項目の行は横並びのまま、ラベルを入力欄の左隣に置く形にしています。
