Vercel 向けの最小構成を配置しています。

入口画面では、ハナビBH と ハッピージャグラーVⅢ と スマスロ Lハナビ と 新ハナビ と スマスロ サンダーV と バーサス リヴァイズ と ハナハナホウオウ ～天翔～ と ゴーゴージャグラー3 と Sファンキージャグラー2 と マイジャグラーV と ネオアイムジャグラーEX の11ページに移動できます。ボタンは縦並びで、スマホでも押しやすい間隔にしています。

各画面では、入力欄の名前を左、入力欄を右に並べています。最後に期待値用の条件と店情報の入力欄があります。ハナビBH、スマスロ Lハナビ、新ハナビでは攻略率を独立した入力欄で使い、初期値は75%です。ハッピージャグラーVⅢ と Sファンキージャグラー2 と マイジャグラーV と ネオアイムジャグラーEX では攻略率は使わず、公表値、チェリー狙い、フル攻略の3択を横一列のラジオボタンで選びます。マイジャグラーV と ネオアイムジャグラーEX の初期状態はチェリー狙いです。ハナハナホウオウ ～天翔～ と ゴーゴージャグラー3 では攻略率は使わず、公表の機械割や出玉率をそのまま期待値に使います。見出しの右隣に「期待値の計算に使用」と表示するページでは、その内容を機械割計算へ反映します。その下の店情報では貸枚数は46、交換率は5.0が初期値です。店情報では貸枚数と交換率の入力欄を短め、現金投資は少し広めにしてあります。貸枚数の右に1枚あたりの貸し単価、交換率の右に1枚あたりの交換単価、現金投資の右に換金ギャップ損をリアルタイム表示します。

各ページの入力内容は、その端末のブラウザ内へ自動で保存します。同じ端末、同じブラウザで開き直したりリロードしたりしても、前回の入力内容を戻せます。さらに、保存データとして保存1から保存5までの5枠を用意し、保存データを押すとその枠の内容を読み込みます。保存データ名は各枠の下の入力欄で変更でき、空欄のときは従来どおり保存1から保存5までの名前を表示します。画面を開いた時点で、保存済みの枠があれば一番小さい番号の保存データを自動で選び、保存済みの枠がない場合は保存1を選びます。保存データを1つだけ選んでいるときは、内容を入力した時点でその枠へ自動保存します。保存データは複数選択でき、複数選んだときは数値入力を合算した内容を入力欄へ表示し、そのまま推測できます。複数選択中は合算表示として扱い、個別の保存データへ合算値を自動上書きしません。選択中の保存データが1つだけのときに同じ保存データをもう一度押しても、未選択には戻しません。複数選択を外す場合も、最後の1つは選択したまま残します。未保存の枠を押したときは初期状態の入力内容を表示します。貸枚数、交換率、攻略率、打ち方の選択は合算せず、保存されている値を引き継ぎます。推測結果そのものは保存せず、入力だけを残します。クリアボタンは保存データを1つだけ選んでいる場合はその保存データと名前を初期状態へ戻し、複数選択中は合算表示を解除して保存データを1つ選んだ状態に戻します。全てクリアボタンは保存1から保存5までの全保存データ、名前、表示中の入力内容を初期状態へ戻し、その後は保存1を選んだ状態にします。スマスロ Lハナビ と 新ハナビ と スマスロ サンダーV と バーサス リヴァイズ では、入力モードの切替状態も端末内へ保存します。

名前にジャグラーを含むページでは、小役入力の近くに3行構成のカウンターを表示します。1行目には横幅を広げた押下確認用メーターと、現在のブドウ数を4桁で示すデジタル表示風の数値を並べています。メーターは5段で構成し、ブドウのボタンを押すたびに帯が1本ずつ追加され、最大5本まで残しながら、それぞれが60秒かけて右端から左へ向かってなめらかに空になります。ブドウ時の帯の色は押すたびに順番で切り替わり、同じ色が続かないようにしています。ブドウ数のデジタル表示も、直前のブドウ押下で使われた帯と同じ系統の文字色へ切り替わるため、最後に押された色を数字側でも確認できます。単独REGのボタンを押したときは、メーターと全体演出だけを灰色系にし、ブドウ数の文字色は変えません。さらに、ボタンを押した瞬間にはカウンター全体へ同系色のフラッシュと短い揺れを入れ、押せたことを周辺視野でも把握しやすくしています。2行目には左にブドウ-1、右に横幅を広げたブドウ+1ボタン、単独REGの設定差がある Sファンキージャグラー2、マイジャグラーV、ネオアイムジャグラーEX では3行目に左に単独REG-1、右に横幅を広げた単独REG+1ボタンを表示します。4つのボタンの左右端は上段のメーターとカウンターの外端にそろえ、ブドウの2つのボタンは従来のおよそ2倍の縦幅にしています。ボタンを押すたびに通常時のブドウ数だけを1つ増減し、自動保存の対象にもなります。ブドウ-1では0未満にならないようにしています。スマホで小役カウンターとして押しやすいように、ブドウ用ボタンは推測ボタンよりさらに大きい緑系のボタンにしています。単独REG+1はBAR図柄をイメージした灰色系のボタン、単独REG-1は「-1」だけの小さめ表示にしています。

スマスロ Lハナビ、新ハナビ、スマスロ サンダーV、バーサス リヴァイズのユニメモ画像読み取りは、公式アプリが生成した縦長画像1枚と、ユーザーが画面を分けて撮った複数画像の両方を同じ画像選択から受け付けます。複数画像は選択順にGeminiへ渡し、機種ごとの表示順、黄色い見出し、前後の項目で区分を判別します。同じ部分が重なって写っている場合は同じ項目として扱い、回数を足しません。見出しも前後項目もなく区分が判別できない項目は入力しません。画像は最大8枚まで、1枚10MB以内、合計20MB以内です。

推測ボタンを押すと、入力値から実践G数や各確率を計算して結果を表示します。ボーナス確率の推測では、開始前の値を差し引かず、現在のG数、BIG、REGの累計値をそのまま使います。小役やBIG中、REG中などの推測では、ユーザーの実践分だけを分母に使います。BIG中の項目はBIG中の消化G数を分母に使い、REG中の項目はREG中の消化G数を分母に使います。

推測結果では、総合推測の割合を最初に表示し、その下の実践期待値の表で実践G数、機械割、各設定の期待値、推測割合、推測期待値、およびその合計期待値を表示します。合計行の機械割には、各設定の機械割を推測割合で重み付けした期待機械割も表示します。さらに、合計の推測期待値をもとに700Gあたりで換算した期待時給も表示します。期待時給は「期待時給 1399.6円 / h」のように、名前の右隣へ横並びで表示します。攻略率を使うページでは、機械割の見出し下に攻略率を表示し、通常時の機械割と完全攻略の機械割の間で値を変えます。攻略率を使わないページでは、公表値や選択した打ち方の機械割をそのまま使います。円換算は交換率から自動で変わり、現金投資額ぶんの換金ギャップ損も差し引いています。

ハナビBH では、ボーナス、通常時小役、花火チャレンジ中、花火ゲーム中、BIG中について、各項目ごとの推測値を表示します。合算と2役合算は表示だけで、総合推測には含めていません。

ハッピージャグラーVⅢ では、ボーナスと通常時小役のブドウについて、各項目ごとの推測値を表示します。ボーナス合算は表示だけで、総合推測には含めていません。期待値計算は攻略率ではなく、公表値、チェリー狙い、フル攻略の選択によって機械割を切り替えます。ブドウが未入力のときは、ブドウを0個として扱わず、総合推測から外してボーナス確率のみで推測します。

スマスロ Lハナビ では、ボーナス、通常時小役、花火チャレ中、花火ゲーム中、BIG中、REG中、REGピース画面について、各項目ごとの推測値を表示します。通常時小役では風鈴Aと風鈴Bをそれぞれ総合推測に使い、風鈴合算は表示だけにしています。ボーナス合算も表示だけで、総合推測には含めていません。入力モードは通常とユニメモの2種類で、初期状態はユニメモです。ユニメモ画像を選ぶと、サーバー側でGeminiを使って総プレイ数、総BB回数、総RB回数、通常時小役、RT中小役、BB中小役、RB中小役、RB終了画面ピース表示回数を読み取り、読み取れた項目だけを入力へ反映します。Geminiの接続には `.env.local` の `GEMINI_API_KEY` を使い、モデルは `GEMINI_MODEL` が未指定の場合 `gemini-2.5-flash-lite` を使います。画像解析は機種追加を前提に、Geminiへ送る共通処理と機種ごとの読み取り項目を分けています。花火チャレ、花火ゲーム、BIG中、REG中だけは切替に応じて入力形式が変わり、ユニメモでは回数と確率を直接入力し、内部では回数と確率から必要な消化G数を組み立てて推測します。REGピース画面は切替に関係なく、2回以下、3～4回、5回以上ごとに試行回数と発生回数を入力でき、設定1は出現なし、設定2、設定5、設定6は同じ出現率として推測に使います。ユニメモ画像から読み取ったRB終了画面ピース表示回数は、2回以下の試行回数と発生回数へ同じ数を入れます。スペック表にもREGピース画面の出現率を表示します。

新ハナビ では、ボーナス、通常時小役、花火チャレ中、花火ゲーム中、BIG中、REG中、REGピース画面について、各項目ごとの推測値を表示します。通常時小役では風鈴Aと風鈴Bをそれぞれ総合推測に使い、風鈴合算は表示だけにしています。通常時小役の入力欄には氷A、チェリーA2、チェリーBもあります。ボーナス合算も表示だけで、総合推測には含めていません。入力モードは通常とユニメモの2種類で、初期状態はユニメモです。ユニメモ画像から、基本情報、通常時小役、RT詳細、BB中詳細、RB中詳細を読み取り、読み取れた項目だけを入力へ反映します。花火チャレ、花火ゲーム、BIG中、REG中だけは切替に応じて入力形式が変わり、ユニメモでは回数と確率を直接入力し、内部では回数と確率から必要な消化G数を組み立てて推測します。REGピース画面は切替に関係なく従来どおりで、0~4回時、5~6回時、7回以上時ごとに試行回数と発生回数を入力でき、入力欄は3桁程度の短めの幅にしています。画像どおりの発生率を使って総合推測と各項目別推測へ反映します。

スマスロ サンダーV では、ボーナス、通常時小役、ボーナス中について、各項目ごとの推測値を表示します。通常時小役ではベルAとベルBをそれぞれ総合推測に使い、ベル合算は表示だけにしています。チェリーB、スイカA、スイカBも個別項目として使い、スイカ合算も表示します。ボーナス合算も表示だけで、総合推測には含めていません。入力モードは通常とユニメモの2種類で、初期状態はユニメモです。ユニメモ画像から、基本情報、小役、BB中小役またはボーナス中小役を読み取り、読み取れた項目だけを入力へ反映します。切替で変わるのはボーナス中だけで、ユニメモではベルB回数と確率から必要な消化G数を組み立て、ベルC回数とリーチ目回数も同じ消化G数に乗せて推測します。

バーサス リヴァイズ では、ボーナス、通常時小役、VS CHANCE中、VS GAME中、BIG中、REG中、RB終了時の赤帯について、各項目ごとの推測値を表示します。通常時小役ではベルAとベルBをそれぞれ総合推測に使い、ベル合算は表示だけにしています。スイカAとチェリーBも総合推測に使います。ボーナス合算も表示だけで、総合推測には含めていません。入力モードは通常とユニメモの2種類で、初期状態はユニメモです。ユニメモ画像から、基本情報、小役、RT詳細、BB中詳細、RB中詳細を読み取り、読み取れた項目だけを入力へ反映します。VS CHANCE中、VS GAME中、BIG中、REG中は切替に応じて入力形式が変わり、ユニメモでは回数と確率から必要な消化G数を組み立てて推測します。RB終了時の赤帯は新ハナビの特殊画面と同じ考え方で、通常時、RT2連まで、RT3連以上に分けて、各条件ごとの試行回数と発生回数を入力できます。

ハナハナホウオウ ～天翔～ では、ボーナス、通常時ベル、BIG中小役、BIG後トップランプ、REG中サイドランプ、連チャンサウンド選択率について、各項目ごとの推測値を表示します。攻略率やユニメモ切替はありません。BIG中小役は通常時ベルの下に表示し、スイカ回数とハズレ回数だけを入力し、分母は実践中のBIG回数×24Gを自動表示して推測に使います。BIG後トップランプは青から虹までの5種類の回数を入力し、分母は実践中のBIG回数を使います。計は表示だけで、総合推測には含めていません。REG中サイドランプは青から虹までの5種類の回数を入力し、分母は実践中のREG回数を使います。ランプ系とBIG中小役の入力欄は短めの幅にし、横方向へ詰めて表示します。連チャンサウンド選択率は新ハナビの特殊画面と同じ考え方で、試行回数と発生回数を入力します。

ゴーゴージャグラー3 では、ボーナスと通常時小役のぶどうについて、各項目ごとの推測値を表示します。ユニメモ切替や攻略率はありません。期待値計算には公表の機械割をそのまま使います。ボーナス合算は表示だけで、総合推測には含めていません。ぶどうが未入力のときは、ぶどうを0個として扱わず、総合推測から外してボーナス確率のみで推測します。

Sファンキージャグラー2 では、ボーナス、通常時小役のブドウ、単独REGについて、各項目ごとの推測値を表示します。ユニメモ切替や攻略率はありません。期待値計算はハッピージャグラーVⅢ と同じく、公表値、チェリー狙い、フル攻略の3択で機械割を切り替えます。ボーナス合算は表示だけで、総合推測には含めていません。ブドウと単独REGはどちらも未入力なら総合推測から外し、入力された項目だけを使って推測します。単独REGは回数のみを入力し、分母には実践ゲーム数を使います。

マイジャグラーV では、ボーナス、通常時小役のブドウ、単独REGについて、各項目ごとの推測値を表示します。ユニメモ切替や攻略率はありません。期待値計算はハッピージャグラーVⅢ と同じ考え方で、公表値、チェリー狙い、フル攻略の3択で機械割を切り替え、初期状態はチェリー狙いです。ボーナス合算は表示だけで、総合推測には含めていません。ブドウと単独REGはどちらも未入力なら総合推測から外し、入力された項目だけを使って推測します。単独REGは回数のみを入力し、分母には通常時の実践G数を使います。

ネオアイムジャグラーEX では、ボーナス、通常時小役のブドウ、単独REGについて、各項目ごとの推測値を表示します。ユニメモ切替や攻略率はありません。期待値計算は公表値、チェリー狙い、フル攻略の3択で機械割を切り替えます。初期状態はチェリー狙いです。ボーナス合算は表示だけで、総合推測には含めていません。ブドウと単独REGはどちらも未入力なら総合推測から外し、入力された項目だけを使って推測します。単独REGは回数のみを入力し、分母には実践ゲーム数を使います。

各項目ごとの推測値は、下のスペック表と同じまとまりごとの表で整理しています。各項目の表では、1列目の見出しにグループ名を入れ、その下に分母となるゲーム数を表示しています。各列見出しは「項目名」と「実測の数（確率）」の2行で小さく表示しています。

設定表は、機種ごとのまとまりに合わせて分けています。ハナビBH の通常時小役には2役合算を表示し、スマスロ Lハナビ と 新ハナビ の通常時小役には風鈴合算を表示します。スマスロ Lハナビ ではREGピース画面の2回以下、3～4回、5回以上の発生率も別表で表示します。スマスロ サンダーV の通常時小役にはベル合算とスイカ合算を表示します。バーサス リヴァイズ の通常時小役にはベル合算を表示し、RB終了時の赤帯は通常時、RT2連まで、RT3連以上を分けて別表で表示します。ハナハナホウオウ ～天翔～ では BIG中小役を通常時ベルの次に表示し、そのあとに BIG後トップランプ、REG中サイドランプ、連チャンサウンド選択率を並べます。新ハナビ の通常時小役には氷A、チェリーA2、チェリーBも表示し、REGピース画面の0~4回時、5~6回時、7回以上時の発生率も別表で表示します。ハッピージャグラーVⅢ と Sファンキージャグラー2 と マイジャグラーV と ネオアイムジャグラーEX では、基本スペック表にBIG、REG、ボーナス合算、公表値を表示し、別表で公表値、チェリー狙い、フル攻略の機械割を並べます。Sファンキージャグラー2 と マイジャグラーV と ネオアイムジャグラーEX ではさらに、通常時小役と単独REGの表を別に表示します。基本スペックには通常の機械割や打ち方ごとの機械割を表示し、確率の分母は小数もそのまま見えるようにしています。

全体の見た目は、スマホで見やすいように横幅と余白を詰めています。スマホでも、2項目と3項目の行は横並びのまま、ラベルを入力欄の左隣に置く形にしています。さらに、スマホ時は外側の枠からはみ出しにくいように余白、文字サイズ、表の見出しの折り返し、ボタンの大きさを微調整していますが、文字や入力欄の基本配置自体は変えていません。
