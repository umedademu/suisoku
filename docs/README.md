# suisoku docs

Vercel 向けの最小構成を配置しています。

ハナビの画面では、入力欄の名前を左、入力欄を右に並べています。
最後に攻略率と店情報の入力欄があり、攻略率は独立した入力欄で初期値は75%です。見出しの右隣に「期待値の計算に使用」と表示しています。その下の店情報では貸枚数は46、交換率は5.0が初期値です。攻略率の入力欄は短めにしてあり、店情報では貸枚数と交換率の入力欄も短め、現金投資は少し広めにしてあります。貸枚数の右に1枚あたりの貸し単価、交換率の右に1枚あたりの交換単価、現金投資の右に換金ギャップ損をリアルタイム表示します。
推測ボタンを押すと、入力値から実践G数や各確率を計算して結果を表示します。BIG中の項目は、BIG中の消化G数を分母に使います。
推測結果では、総合推測の割合を最初に表示し、その下の実践期待値の表で実践G数、機械割、設定1・2・5・6それぞれの期待値、推測割合、推測期待値、およびその合計期待値を表示します。さらに、合計の推測期待値をもとに700Gあたりで換算した期待時給も表示します。機械割の見出し下には攻略率を表示し、機械割はその攻略率に応じて通常時の機械割と完全攻略の機械割の間で変わり、期待値の計算にもその値を使います。円換算は交換率から自動で変わり、現金投資額ぶんの換金ギャップ損も差し引いています。
さらに、ボーナス、通常時小役、BIG中、花火チャレンジ中、花火ゲーム中について、各項目ごとの推測値を表示します。
その8要素をまとめた、設定1・2・5・6の総合推測も表示します。合算と2役合算は表示だけで、総合推測には含めていません。
各項目ごとの推測値も、下のスペック表と同じまとまりごとの表で整理しています。
各項目の表では、1列目の見出しにグループ名を入れ、その下に分母となるゲーム数を表示しています。各列見出しは「項目名」と「実測の数（確率）」の2行で小さく表示しています。
設定表は、基本スペック、通常時小役、BIG中、花火チャレンジ中、花火ゲーム中のまとまりごとに分けていて、通常時小役には2役合算も表示しています。基本スペックには通常の機械割と機械割(完全攻略)を表示し、確率の分母は、整数でも小数点第一位まで表示します。
全体の見た目は、スマホで見やすいように横幅と余白を詰めています。
スマホでも、3項目の行は横3列のまま、ラベルを入力欄の左隣に置く形にしています。
